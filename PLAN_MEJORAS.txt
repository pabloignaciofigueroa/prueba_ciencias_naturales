FASE 3 — SISTEMA DE REACCIONES (notificaciones overlay)
==============================================================
Objetivo: Mostrar REACCIONES CORRECTO o REACCIONES FALLO
          como overlay flotante animado al responder.

Comportamiento:
  - Al acertar: REACCIONES CORRECTO.png aparece desde abajo,
    escala de 0.6→1.1→1, permanece ~1.2s, luego desaparece.
    Se combina con el burst de partículas existente.
  - Al fallar: REACCIONES FALLO.png aparece igual.
    Permanece ~1.5s.

Posición: centrado en pantalla, z-index 200.
          position:fixed, bottom 15%, left 50%, transform translateX(-50%).
Tamaño: max-width 280px, height auto (respetar proporción).
NO se superpone con el contenido de la pregunta porque es overlay.

Implementación React:
  - Estado: reactionImg (null | 'correcto' | 'fallo')
  - Componente <ReactionOverlay> que muestra el PNG con animación.
  - Se dispara en handleAnswer(), desaparece con setTimeout.

==============================================================
FASE 4 — FLUJO DE FALLO: NO AVANZAR, PERMITIR REINTENTO
==============================================================
Objetivo: Tono amigable. Si fallas, la pregunta te espera.

Comportamiento nuevo al fallar:
  1. Mostrar REACCIONES FALLO.png (overlay, ~1.5s).
  2. Hacer "camera shake" — la tarjeta .q-card anima con
     keyframe @cameraShake (translateX + ligero rotate).
  3. Después de ~1.8s: limpiar feedback, selectedOpt,
     y RE-SHUFFLE las opciones de la misma pregunta.
     La pregunta vuelve a estar activa para intentar de nuevo.
  4. NO se suma al contador de incorrectas para la puntuación
     del reintento (solo cuenta la respuesta final correcta).
     ALTERNATIVA SIMPLE: cada intento cuenta; la estrella
     refleja naturalmente cuántas acertó de 5.
     → DECISIÓN: usar la alternativa simple (más honesta).
  5. El botón "Siguiente" NO aparece al fallar.
     Solo aparece "Siguiente" tras acertar.

Tono del mensaje de fallo:
  Reemplazar "💪 ¡Casi! era: X" por mensajes como:
  - "¡Casi! Inténtalo de nuevo 🌟"
  - "¡Tú puedes! Una más 💪"
  - "¡No te rindas, {name}! 🚀"
  (El mensaje desaparece cuando se limpie el feedback.)

Efecto cameraShake CSS:
  @keyframes cameraShake {
    0%,100% { transform: translateX(0) rotate(0deg); }
    15%     { transform: translateX(-10px) rotate(-1.5deg); }
    30%     { transform: translateX(10px) rotate(1.5deg); }
    45%     { transform: translateX(-8px) rotate(-1deg); }
    60%     { transform: translateX(8px) rotate(1deg); }
    75%     { transform: translateX(-4px); }
    90%     { transform: translateX(4px); }
  }
  Duración: 0.55s. Se aplica clase .camera-shake al .q-card.
